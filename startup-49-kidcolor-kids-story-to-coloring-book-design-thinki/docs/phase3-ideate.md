## Chosen Concept: Story-to-Book as Primary + Interest Packs as Fast Track

**Rationale:**

| Factor | Story-to-Book | Interest Packs |
|--------|--------------|----------------|
| Differentiation | ★★★★★ (nothing like it exists) | ★★★ (many competitors) |
| Build complexity | Medium (wizard + voice) | Low (icon grid + slider) |
| Kid engagement | ★★★★★ (authored = attached) | ★★★ (relevant = liked) |
| Parent conversion | ★★★★★ (emotional preview) | ★★★★ (convenient) |
| Teacher channel | ★★★★ (curriculum story) | ★★★★★ (class packs) |
| Time to MVP | 3–4 sprints | 1–2 sprints |

**Decision:**
1. Build **Interest Packs first** (2-sprint MVP, lower risk, validates generation quality)
2. Layer **Story-to-Book** on top (adds wizard + voice to Interest Packs base)
3. **Adventure Builder** enters roadmap at Phase 4 as the subscription retention layer

This sequence means: fast first revenue (Interest Packs), then deep differentiation (Story-to-Book), then lock-in (Adventure Builder subscription).

## Prompt Engineering Templates

### Style System Prompt (all concepts)
```
You are generating children's coloring book pages. Every image must:
- Use ONLY thick black outlines (minimum 3pt line weight) on pure white background
- Contain ZERO shading, gray tones, hatching, or cross-hatching
- Feature simple, age-appropriate scenes a child aged [AGE] can color
- Have clearly enclosed regions (all paths closed — no gaps)
- Be 300 DPI equivalent at 2550×3300px (letter size)
- Contain NO text, words, or letters unless explicitly requested
- Contain ONLY content appropriate for children aged 3-12
```

### Story-to-Book Prompt Template
```
[SYSTEM PROMPT]

Scene [PAGE_INDEX] of [TOTAL_PAGES] from a children's story:
"{HERO_NAME} the [CHARACTER_1] and the [CHARACTER_2] [ACTION] in [SETTING]"

This page shows: {PAGE_SCENE_DESCRIPTION}

Style: Children's coloring book, thick black outlines only, white background,
age [CHILD_AGE] appropriate, printable line art, no color, no shading.
```

### Interest Packs Prompt Template
```
[SYSTEM PROMPT]

A children's coloring book page featuring:
- Main subject: {INTEREST_1} ({CHARACTER_DESCRIPTION})
- Secondary element: {INTEREST_2}
- Setting: environment related to {INTEREST_3}
- Child age: {AGE_RANGE} → line weight {LINE_WEIGHT}pt minimum

Create a cheerful, engaging scene with clearly separated regions for coloring.
```

### Safety Wrapper (mandatory for all prompts)
```
IMPORTANT SAFETY RULES:
- No weapons, violence, blood, or scary content
- No adult themes, romance, or suggestive imagery
- No real people, celebrities, or existing IP characters
- If any unsafe request is detected, generate a safe substitute scene instead
- Output must be appropriate for display in a kindergarten classroom
```

## Prompt Safety Guidelines (Full Spec → see docs/prompt-safety-guidelines.md)

Summary of Layer 2 prompt safety rules:
1. **Character whitelisting:** Only allow characters from the pre-defined wizard grid (dinosaur, unicorn, robot, etc.) — no free-text character names in image prompt
2. **Setting whitelisting:** Only allow settings from the pre-defined grid
3. **Hero name sanitization:** Hero name used only in page headers, never in image prompt
4. **Scene description generation:** System generates scene descriptions from structured wizard data — raw user text never reaches the image model
5. **Age-range enforcement:** Line weight and scene complexity locked to age range
6. **Content category blocking:** Pre-generation blocklist of 200+ keywords
